### __1.SELECT PHASE__
```
各戦闘参加機体はそれぞれ座標移動数と、予約ARTSを決定する。

(座標移動数上限)=(機動)-(積載)(min=1,MAX=3)
(座標移動数下限)=0
```
```
移動後の座標と使用するARTS：No.とARTSの対象を各自座標盤面GUIに入力し、【確定】ボタンを押下する。
この処理を、『ARTSを予約する』と定義する。
(アーツを使用しない場合、ARTS：No.には0を入力する)

その後【表示/更新】ボタンを押下する。
すると、入力待機中(x/y)のxの値に自身が数えられていることを確認できる。

(x=行動予約済機体数)
(y=戦闘参加機体数)
```
```
全戦闘参加機体の移動数が決定されたとき(x=yになったとき)、移動後の座標関係と各戦闘参加機体の予約アーツを公開し、2.ACTION PHASEに移行する。
```
### __2.ACTION PHASE__
```
1：戦闘参加機体は操縦技能【機動制御】の技能行使判定を行い、行動順を決定する。
行動順は速度指数の降順とする。

(速度指数)=(【機動制御】の技能行使Lv)×(機体の機動)

また、優先度：1以上のARTSを予約している機体は、速度指数に関わらず優先して行動順を獲得する。

優先度：1以上のARTSを使用する機体が複数存在する場合は、該当機体の予約ARTSの優先度値を比較し、その降順で行動順を決定する。
```
```
2：速度指数の等しい機体が存在するとき、該当機体は操縦技能【機動制御】の技能対抗判定を行い、技能行使Lvの降順に行動する。

技能行使Lvが同値のとき、両者は再判定を行い、差が生じるまでこれを繰り返す。
```
```
3：戦闘参加機体に行動権：1を付与し、決定された行動順に従ってTURNを与える。
自身のTURNを与えられた機体は、以下の順番で行動を処理する。
```
```
①TURN START
自身のTURNを開始することを宣言する。
その後、②に移行する。
```
```
②TURN ACTION
行動権を消費して予約ARTSを使用するか、あるいは【OVER：RIDE】を行う。
【OVER：RIDE】を行わず、予約ARTSによって選択できる対象が存在しないとき、行動権を消費して③に移行する。
行動権が尽きるか、もしくは任意で③に移行する。
```
```
【OVER：RIDE】
2.SELECT PHASEで宣言した予約ARTS/予約対象を、以下の手順で変更できる。

❶変更先のARTS/対象を指定する。
❷操縦技能【危機感知】および変更先ARTSの依存技能をそれぞれ判定する。  
❸判定結果を合算し、x とする。  

(x) = (【危機感知】の技能行使Lv) + (変更先ARTSの依存技能行使Lv)  

x≧13のとき、予約ARTS/予約対象をそれぞれ変更先に上書きする。
x<13のとき、行動権を消費して③TURN ENDに移行する。

※このとき、変更先ARTSの故障判定は通常通り行われる。  
※優先度：1以上のARTSを使用している場合、【OVER：RIDE】は行えない。
```
```
③TURN END
自身のTURNを終了することを宣言する。
戦闘参加機体のすべてのTURNが終了したとき、3.END PHASEに移行する。
```
### __3.END PHASE__
```
2.SELECT PHASEに移行する。
敵対機体が戦闘参加機体に存在しない場合、戦闘を終了する。
1〜3のPHASEの流れ、その一纏まりの単位を、『1WAVE』と定義する。
```
